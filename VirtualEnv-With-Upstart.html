Upstart With Python VirtualEnv

  • Published: 29 Nov 2012
Share

I thought I would write this very short blog article to help anyone who would who wants to launch a uwsgi server that refers to a python script in a virtual environment via upstart.

In order to run a uwsgi server that runs a python script that in virtualenv, the following switches must be defined:

  1. pythonpath - this should be set to the folder where your script lives. Uwsgi uses this switch to define the PYTHONPATH environment variable for the script to be executed.
  2. virtualenv - this must be set the folder where your virtual environment is defined. It is normally ~/.virtualenvs/<name of virtual environment>. Uwsgi uses this switch to define the PYTHONHOME environment variable for the script to be executed.
  3. file - this is the file that will be run by uwsgi
